NullAsh Documentation Naming Convention (Standard v1.0)
This system ensures traceability and professionalism across all NullAsh projects.

1. Naming Structure

The format follows a hierarchical code: `[ORG]-[PROJ]-[DOC]-[NUM]`

• [ORG] (Organization):

  • `NS`: NullAsh Studio (Software, AI, Game Dev)

  • `NR`: NullAsh Robotics (Hardware, Mechanical, Embedded)

• [PROJ] (Project Code):

  • `CRAI`: CodeReviewerAI

  • `HOKH`: Hookah Maker

  • `CONS`: Custom Console (RISC-V)

  • `HISO`: Hisoka System (Game Mechanics)

• [DOC] (Document Type):

  • `DD`: Design Document (High-level architecture, specs)

  • `TR`: Test Report (Verification, pass/fail results)

  • `LOG`: Engineering Log (Daily progress, messy notes)

  • `MAN`: Manual / User Guide

  • `SD`: Schematic / Diagram

• [NUM] (Sequence Number):

  • `001`, `002`, etc. (Always use three digits for sorting).

---

2. Examples

• `NS-CRAI-DD-001`: The primary architecture plan for CodeReviewerAI.

• `NR-HOKH-TR-005`: The fifth technical test for the Hookah Maker thermal sensor.

• `NR-CONS-LOG-042`: A specific daily entry regarding the Console's GPU logic.

3. Version Control (Optional suffix)

If a document is updated, add `_Rev[Letter]` to the end:

• `NS-CRAI-DD-001_RevB`: The second version of the Design Document.
